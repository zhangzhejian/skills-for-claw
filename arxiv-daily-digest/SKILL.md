# Arxiv 每日论文摘要

从 arxiv 获取最近论文，做**高价值筛选**，生成面向研究者和工程师的中文深度速览。

## 核心原则

**你不是翻译机器，你是研究顾问。** 读者是有背景知识的研究者/工程师，他们需要的不是论文列表，而是：
1. 今天有什么值得我花时间读的？为什么？
2. 这个方向正在发生什么变化？
3. 哪些工作对我的项目有直接参考价值？

## 工作流程

### 1. 理解用户需求

用户会指定研究方向或主题。帮他们映射到合适的 arxiv 分类和搜索关键词。

常见的 arxiv CS 分类参考：
- `cs.AI` - 人工智能
- `cs.CL` - 计算语言学（NLP）
- `cs.CV` - 计算机视觉
- `cs.LG` - 机器学习
- `cs.CR` - 密码学与安全
- `cs.SE` - 软件工程
- `cs.RO` - 机器人
- `cs.IR` - 信息检索
- `cs.MA` - 多智能体系统
- `stat.ML` - 机器学习（统计）

### 2. 获取论文

使用 `scripts/fetch_arxiv.py` 脚本获取论文：

```bash
python ~/.claude/skills/arxiv-daily-digest/scripts/fetch_arxiv.py \
  --categories "cs.AI,cs.CL" \
  --keywords "LLM,agent,reasoning" \
  --max-results 100 \
  --hours 72
```

参数说明：
- `--categories`：逗号分隔的 arxiv 分类
- `--keywords`：逗号分隔的关键词（会在标题和摘要中搜索）
- `--max-results`：每个查询最多返回的结果数（默认 100）
- `--hours`：回溯时间范围，默认 24 小时。**如果 24 小时内论文数量太少（< 5 篇），自动扩展到 72 小时**

脚本会输出 JSON 格式的论文列表到 stdout。

### 3. 深度筛选（核心步骤）

拿到论文后，**不要直接生成报告**。先做严格筛选：

**阅读每篇论文的 abstract，回答以下问题：**
1. 这篇论文解决的是真问题还是人造问题？
2. 方法上是否有实质性新意（vs 换了个数据集/加了个模块的增量工作）？
3. 实验是否有说服力（baseline 是否 strong、ablation 是否充分）？
4. 对从业者是否有直接可用的 insight 或方法？

**筛选标准（从严）：**
- 真正值得精读的论文（Must Read）：通常一天只有 0-3 篇
- 值得了解的论文（Worth Knowing）：5-8 篇
- 其余的直接不报告，不要凑数

**常见低价值信号（遇到请降权或跳过）：**
- 仅在小数据集上刷点的微小改进
- 没有对比当前 SOTA baseline 的实验
- 标题含 "A Survey of..." 但只是文献堆砌
- 仅重新组合已有模块、没有新 insight
- Abstract 全是 buzzword 但说不清楚具体贡献

### 4. 生成报告

**报告格式（注意：不是论文列表，是分析报告）：**

```markdown
# [方向名称] — 最新论文速览

> 检索范围：YYYY-MM-DD 至 YYYY-MM-DD | 分类：[categories] | 关键词：[keywords]
> 共检索 N 篇，筛出 M 篇值得关注

---

## 🔥 Must Read（最值得精读的 1-3 篇）

### 1. [一句话概括这篇论文做了什么、为什么重要]

**[论文标题]**（[arxiv_id]）— [一作姓名] et al.
[链接]

[2-4 句话的深度分析，需要包含：]
- 解决什么问题：不是复述 abstract，而是用自己的话说清楚为什么这个问题重要
- 关键方法：具体说出技术创新点是什么，用一句话让读者理解 "哦，原来是这么做的"
- 为什么值得读：相比已有工作（如果你知道的话）好在哪里，或者揭示了什么 surprising 的 finding
- 对谁有用：明确说出 "如果你在做 X，这篇论文的 Y 方法/发现值得参考"

---

## 📌 Worth Knowing（值得了解的 5-8 篇）

> 这些论文不一定需要精读全文，但了解核心 idea 有价值。

| # | 论文 | 核心 idea（一句话） | 为什么值得关注 |
|---|------|---------------------|----------------|
| 1 | **标题缩写**（arxiv_id）[链接] | ... | ... |
| 2 | ... | ... | ... |

---

## 🔍 趋势与观察

[3-5 句话的趋势分析，需要有洞察而非泛泛而谈。好的趋势分析示例：]
- "本周连续出现 3 篇将 RL 用于 agent 工具调用训练的工作（KARL, WebFactory, X），说明社区正在从 SFT→RL 范式迁移，且焦点从对齐转向能力获取"
- "值得注意的是 X 和 Y 两篇论文从完全不同的角度得到了相似的结论：Z，这可能暗示..."

[不好的趋势分析：]
- "AI 领域发展迅速，本周有很多值得关注的工作"（太空泛）
- "RL 越来越重要"（没有具体证据支撑）
```

### 写作要点

1. **说人话**：避免 "本文提出了一种新颖的方法..."，而是 "这篇论文的 insight 是：如果把 X 当成 Y 来处理，就可以..."
2. **有态度**：如果一篇论文的实验不够 convincing，直接说 "实验只在 X 上做了，需要更多验证"，不要客气
3. **给 context**：尽量关联到读者可能知道的已有工作，比如 "类似 DPO 对 RLHF 的简化，这篇论文对 X 做了类似的事"
4. **实用导向**：每篇论文都要回答 "so what"——对做工程/研究的人来说意味着什么
5. **宁缺毋滥**：如果某天确实没有高质量论文，就说 "今天这个方向没有特别突出的工作"，不要硬凑
6. **保留原标题**：论文标题保留英文原文，不翻译标题

### 输出

将报告保存为 Markdown 文件，文件名格式 `arxiv_digest_YYYYMMDD_[主题简称].md`。
