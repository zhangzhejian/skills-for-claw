# Arxiv 每日论文摘要

帮助用户获取 arxiv 上最近 24 小时内特定方向的最新论文，并生成结构化的中文报告。

## 工作流程

### 1. 理解用户需求

用户会指定：
- **研究方向**：如 cs.AI, cs.CL, cs.CV, cs.LG 等 arxiv 分类
- **关键主题**：如 "LLM agent", "code generation", "multimodal" 等关键词

如果用户只给了自然语言描述（如"大语言模型相关的"），帮他们映射到合适的 arxiv 分类和搜索关键词。

常见的 arxiv CS 分类参考：
- `cs.AI` - 人工智能
- `cs.CL` - 计算语言学（NLP）
- `cs.CV` - 计算机视觉
- `cs.LG` - 机器学习
- `cs.CR` - 密码学与安全
- `cs.SE` - 软件工程
- `cs.RO` - 机器人
- `cs.IR` - 信息检索
- `stat.ML` - 机器学习（统计）

### 2. 获取论文

使用 `scripts/fetch_arxiv.py` 脚本获取论文。运行方式：

```bash
python ~/.claude/skills/arxiv-daily-digest/scripts/fetch_arxiv.py \
  --categories "cs.AI,cs.CL" \
  --keywords "LLM,agent,reasoning" \
  --max-results 100 \
  --hours 24
```

参数说明：
- `--categories`：逗号分隔的 arxiv 分类
- `--keywords`：逗号分隔的关键词（会在标题和摘要中搜索）
- `--max-results`：每个查询最多返回的结果数（默认 100）
- `--hours`：回溯时间范围，默认 24 小时

脚本会输出 JSON 格式的论文列表到 stdout。

### 3. 生成报告

拿到论文数据后，按以下格式生成中文 Markdown 报告：

```markdown
# Arxiv 每日论文摘要

> 生成时间：YYYY-MM-DD HH:MM
> 检索范围：最近 24 小时
> 研究方向：[列出分类]
> 关键主题：[列出关键词]
> 共检索到 N 篇相关论文

---

## [方向名称1]（如：大语言模型）

### 1. [论文标题中文翻译]

- **原标题**：English Title
- **作者**：Author1, Author2, ...
- **链接**：https://arxiv.org/abs/XXXX.XXXXX
- **摘要总结**：用 2-3 句话概括论文的核心内容和方法
- **创新性评价**：对论文的创新点进行简要评价，包括：
  - 解决了什么问题
  - 方法上有什么新意
  - 潜在影响力如何

---

### 2. [下一篇论文]
...

## [方向名称2]

...

---

## 总结

对当天的论文趋势做一个简短的总体观察，指出值得特别关注的论文。
```

### 要点

- **分组**：按用户指定的研究方向/主题进行分组，而不是按 arxiv 分类。一篇论文可能同时属于多个方向，选择最相关的那个分组放置。
- **摘要总结**：不要直接翻译摘要，而是提炼核心贡献。读者是研究人员，假设他们有相关背景知识。
- **创新性评价**：基于论文摘要给出客观评价。关注方法的新颖性、问题的重要性、以及与现有工作的区别。如果看起来是增量改进就直说，如果有突破性思路也要指出。
- **语言**：报告全部使用中文，但保留论文原标题（英文）。
- **排序**：每个分组内，按创新性和相关度排序，最值得关注的放前面。
- **去重**：如果同一篇论文被多个关键词匹配到，只出现一次。
- **输出**：将报告保存为 Markdown 文件，文件名格式 `arxiv_digest_YYYYMMDD.md`。
